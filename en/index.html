Open source distribute control system
The goal of the project is to create systems from simple "DIY" to distributed industrial ones, on the basis of unified concepts.

The project is open source, both for "hardware" and for software.

Materials will be published step by step, consistently explaining the ideas of building a system that we will jointly create.

The system is built from modules of two levels: L0 and L1.

The module is a set of hardware and software parts designed to solve specific problems.
•
L0 - modules for working with sensors / actuators.
•
L1 - modules for operation as a node of the system.
The system should be universal, scalable and as close as possible to the user's requirements. Initially, the system is designed as a network. So need to provide the necessary protection of information or data encryption.
 
Selecting a microcontroller for L0 modules.
At the current moment, the so-called "Blue Pill" has become the most widely used as an starting module. Let's take it as a basis for us projects. This is a microcontroller STM32F103C8T6.

The articles will not consider the microcontroller architecture and low-level programming, this information is enough on the Internet. Will be offered "firmware" for specific modules.
To start making prototypes, the following minimum set of "hardware" will be used:
Blue Pill - STM32duino  - 2 pcs.
UART-USB adapter - 2 pcs.
Set of wires for solderless installation.

Prototypes of the L0 modules use inexpensive sensors / actuators designed for arduino.
